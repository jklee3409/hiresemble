# `docs` 디렉터리 안내

## 디렉터리 목적

이 디렉터리는 Hiresemble의 제품 계약, 파생 구현 설계와 Codex 작업 규칙을 분리해 관리한다. `spec/`는 무엇을 만들지, `design/`은 계약을 어떤 모듈과 순서로 구현할지, `agent-rules/`는 저장소에서 어떻게 안전하게 작업하고 기록할지 정의한다.

## 주요 파일 및 하위 디렉터리

| 경로                           | 역할                                                  |
| ------------------------------ | ----------------------------------------------------- |
| [`spec/`](spec/)               | 기능, API, DB, 페이지, 기술 스택의 기준 명세          |
| [`design/`](design/)           | 전체 시스템 설계, 명세 정합성 이슈와 단계별 구현 계획 |
| [`agent-rules/`](agent-rules/) | 작업 절차와 백엔드·프론트엔드·인프라 세부 규칙        |
| [`progress.md`](progress.md)   | 문서 영역 전체의 변경 상태와 검증 이력                |

## 다른 디렉터리와의 의존 관계

- [`../AGENTS.md`](../AGENTS.md)는 작업 유형에 따라 이 디렉터리의 세부 문서를 읽도록 라우팅한다.
- `backend/`와 `frontend/` 구현은 `spec/`의 계약을 따른다.
- `design/`은 `spec/`을 대체하지 않고 기능·DB·API·페이지 연결과 구현 선행 관계를 제공한다.
- 각 코드 디렉터리의 `index.md`는 필요한 상세 문서에 상대 경로로 연결한다.

## 변경 시 주의사항

- 목표 계약(`spec`), 파생 구현 설계(`design`), 작업 절차(`agent-rules`)를 한 문서에 섞지 않는다.
- 명세 변경은 API, DB, 페이지, 기술 문서 사이의 교차 영향을 확인한다.
- 설계 문서의 권장 해결안을 명세 승인 없이 확정 계약이나 구현 완료로 기록하지 않는다.
- 현재 구현 여부는 명세에 완료처럼 적지 않고 각 `progress.md`에서 관리한다.
- 상위 문서는 개요만 유지하고 세부 내용은 하위 문서로 연결한다.
